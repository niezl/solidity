⼀、错误抛出⽅式
1. require: ⽤于输⼊验证和访问控制。例如，确保输⼊值 i ⼩于或等于10,否则抛出错误信息“i is
greater than 10”。
2. revert: 适⽤于多层if语句中的条件检查。例如,如果 i ⼤于10,则使⽤ revert 抛出“i is
greater than 10”。
3. assert: ⽤于检查始终应为真的条件。例如，断⾔某状态变量 num 始终等于123,否则表明合约存
在bug。
⼆、⾃定义错误
• ⾃定义错误⽤于节省gas。例如,声明⼀个错误类型 MyError ，并在条件 i ⼤于10时使⽤
revert 与 MyError 抛出错误，同时记录调⽤者地址和输⼊值。
三、状态变量与Gas退款
• 错误抛出将撤销所有状态变更,并可能退还未⽤完的gas。例如,如果函数在状态变量更新后因输
⼊检查失败⽽抛出错误，则所有更改将被撤销。
