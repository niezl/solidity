1. 事件的定义和⽤途
• 事件在区块链上写⼊数据。
• 数据⽆法被智能合约检索。
• 事件的主要⽬的是记录某些事情发⽣的⽇志。
• 作为存储数据的状态变量的廉价替代⽅案。
2. 声明事件
• 使⽤ event 关键字声明事件。
• 事件名称和参数类型的定义。
• ⽰例：声明⼀个包含字符串和 uint 类型参数的事件。
3. 记录事件
• 使⽤ emit 关键字记录事件。
• 提供实际参数并将其记录到区块链。
• ⽰例：记录包含字符串和 uint 类型参数的事件。
4. 事务性函数
• 记录事件的函数不是只读函数。
• 为什么记录事件需要事务性函数。
5. 索引事件
• 使⽤ index 关键字为事件参数创建索引。
• ⽰例：声明和记录带索引参数的事件。
• 通过索引快速搜索事件⽇志。
6. 索引的限制
• 最多可以为三个参数创建索引。
• 如果事件参数超过三个，不能为所有参数创建索引。
