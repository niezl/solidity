1. 映射的限制：
◦ 映射(mapping)在Solidity中不能直接获取其⼤⼩,也⽆法直接遍历其所有元素。
2. 可迭代映射的构建：
◦ 为了能够遍历映射并获取其⼤⼩，需要额外维护⼀个记录所有键的数组和⼀个标记键是否已插
⼊的映射。
3. 主要数据结构：
◦ 映射 balances ：⽤于存储地址对应的余额（ address 到 uint 的映射）。
◦ 数组 keys ：存储所有已插⼊的键（地址），使映射可遍历。
◦ 映射 inserted ：标记⼀个键（地址）是否已被插⼊，防⽌重复记录。
4. 核⼼功能实现：
◦ 设置余额：通过 set 函数更新 balances 映射，并在 keys 数组中记录新键，如果该键未被
之前记录。
◦ 获取映射⼤⼩： getSize 函数返回 keys 数组的⻓度，即映射的⼤⼩。
◦ 访问特定元素：通过索引访问 keys 数组，再通过此地址在 balances 映射中查找对应的余
额。
