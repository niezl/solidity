1. 课程介绍
• ⽬标：理解并掌握在Solidity中验证签名的过程
• 涵盖内容：消息签名、消息哈希、签名验证
2. 签名验证的基本步骤
1. 消息准备：准备需要签名的消息。
2. 消息哈希：对消息进⾏哈希处理。
3. 消息签名：使⽤钱包离线签名消息哈希。
4. 签名验证：在智能合约中使⽤ ecrecover 函数验证签名。
3. 编写签名验证函数
• 函数定义：创建⼀个名为 verify 的函数，⽤于验证签名。
• 输⼊参数
◦ 签名者地址(address)
◦ 消息(string)
◦ 签名(bytes)
4. 具体实现步骤
1. 消息哈希
◦ 使⽤ keccak256 对消息进⾏哈希处理。
2. ⽣成Ethereum签名消息哈希
◦ 对消息哈希进⾏⼆次哈希处理,⽣成Ethereum签名消息哈希。
3. 恢复签名者地址
◦ 使⽤ ecrecover 函数恢复签名者地址。
◦ ⽐较恢复的地址和输⼊的签名者地址是否相等。
5. 内部辅助函数
• getMessageHash 函数
◦ 输⼊:消息(string)
◦ 输出:消息哈希(bytes32)
• getEthSignedMessageHash 函数
◦ 输⼊:消息哈希(bytes32)
◦ 输出:Ethereum签名消息哈希(bytes32)
• recover 函数
◦ 输⼊:Ethereum签名消息哈希(bytes32),签名(bytes)
◦ 输出:签名者地址(address)
• splitSignature 函数
◦ 输⼊:签名(bytes)
◦ 输出:签名的r、s、v值(bytes32, bytes32, uint8)
6. 验证签名的完整代码⽰例
• 实现完整的智能合约，包含所有上述函数。
7. 编程作业
• 任务描述
◦ 编写⼀个智能合约，包含 verify 函数以及所有必要的内部函数。
◦ 部署智能合约,并使⽤MetaMask签名消息。
◦ 验证签名的有效性。
• 要求：
◦ 提供合约代码。
◦ 提供签名验证的测试步骤和结果。
